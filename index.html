What are SVGs?

What are SVGs?

Scalable Vector Graphic (SVG) is a generalized graphics markup language

  • icons
  • complex images
  • data visualizations
  • animated pictures
  • interactive applications

SVGs or Scalable Vector Graphics are an XML-based graphics markup language

  • it has support for two-dimensional graphics, interactivity and animation
  • it's an open standard developed by the w3c and is currently undergoing an update to SVG version 2.0
See the Pen Responsive Huggy Laser Panda Factory by Sarah Drasner (@sdras) on CodePen.

Here's a great example of what you can do with an svg. It is titled Responsive Huggy Laser Panda Factory by Sarah Drasner. It is an interactive, animated and responsive industrial looking toy panda factory. There is so much amazing potential with SVG and it is being used a lot more on the web now.

Accessibility Tree

Image from WICG AOM

Let's look at the accessibility tree.

Users of assitive technology are not really interacting directly with the page content. They are presented with the page's accessibility tree.

Native Element

Image from WICG AOM

Native HTML elements are implicitly mapped to accessibility APIs. For example, an <img> element will automatically be mapped to an accessibility node with a role of "image" and a label based on the alt attribute.

ARIA - Accessible Rich Internet Applications

  • role (checkbox)
  • state (checked)
  • property (e.g. aria-label="label")

So, ARIA was created by the W3C and it provides a standard that allows developers to set roles, states and properties on html elements so that they can be mapped properly to the Accessibility API.

Please note - Just because you change the name of a div to a checkbox does not make it function like a checkbox. You are responsible to wire up all of the functionality of the checkbox.

Developer Tools

Too see how things are mapped via the Accessibility Tree, open up the browser dev tools and look for the accessibilty tab.

The example you see here is Safari, Chrome and Firefox. Each is inspecting the exact same SVG but they all map it to a different role. This is important to remember for later.

This code-theme logo looks pretty basic, so let's start off with the typical img + src (the svg) + the alt attribute. This example works everywhere, but....

SVG as <img> src - zombie Safari bug

Zombie Safari bug

there has been a zombie Safari bug (affecting desktop and mobile) that was fixed in 2016 that resurfaced again in 2020 and is now fixed...again: Zombie Safari bug. In order to be certain the image is supported in Safari, you should add role="img" to the element.

SVG as <img> src - linked

Now, if we want to add a link around the image from the previouse example, all I needed to do was add a link around the image element and update the alt text to reflect that it goes to the home page.

Inline SVG - unlinked

In order to animate or change colors on the fly in the image, we need to inline the SVG directly in the HTML.

To make an inline svg accessible, we need to add the title element inside of it with a short title.

NOTE - Any time you use the title in an svg, it will give users a titled tooltip on hover.

Unfortunately, there are issues with both Firefox + NVDA AND Safari + VoiceOver. Firefox maps the SVG to diagram and won't recognize it AND Safari maps the SVG to nothing and won't recognize it.

Inline SVG - unlinked - fixed

To work around the Firefox and Safari bugs, we need to add role="img" to the SVG so they will map it to an image/graphic role

Inline SVG - unlinked - title and description

Let's say we modified the colors of the svg for a holiday or something. In order to add a description to account for the logo change, we can use the desc element inside of the svg. Starting from where we left off with the previous example...let's add the desc element.

For some reason, I couldn't find one browser/screen reader combo that would read the description

Inline SVG - unlinked - title and description - fixed

So, in order to get this to work, I needed to add aria-labelledby="" with the id's of both the title and desc

Inline SVG - linked - aria-label

Let's say we want to link the inline svg. Unfortunately, just wrapping an anchor around the svg did not work as expected, so..

Needed to add aria-label="and the alternative text here" to the link and aria-hidden="true" to the svg

NOTE - IF you still support ie11, you need to add focusable=“false” to hidden SVGs because otherwise, it will add a tab stop

Inline SVG - Animation

Another fun thing we can do with svgs is animate them.

I know Val Head is covering accessible animations for this conference, but a few quick things to note here are:

animations must be stoppable if they last beyond 5 seconds.

Inline SVG - Animation

in this case, the animation is less than 5 seconds and I added the js function to check the system setting and if it's not set, play the animation, otherwise, it does nothing. The function also watches for the setting to change as well.

Next Example: Icons

Now that we covered the logo from, what I hope, is every possible angle...Let's look at the site again for some more SVGs to look at.

There are a bunch of icons used in this site (the location, coffee beans and cart links in the header, phone and twitter icons in the footer) AND almost each one is implemented in a different way because they have a different context on the page.

Let's dig in.

Icon - decorative - linked, not linked and button

For starters, let's look at the decorative images. We have three examples here: Linked, unlinked and a button. In each case, the icon is redundant to the text next to it.

So,to hide these decorative icons from screen readers, we can add aria-hidden to the SVG and it worked for each variation.

Icon - content - not linked

The next icon we'll address is the phone image in the footer next to the phone number.

Since this image provides context to the content, we should give it alternative text.

Depending on the system generating your icon file, you may or may not need to add a title to the svg, BUT please make sure the text in it makes sense in the context.

Next, we need to add role="img" to the svg.

NOTE - Using a title in these cases with content icons is good because, in the event that the icon is vague, it will give users a titled tooltip on hover.

Icon - content - linked

This video clip shows the navigation text in the header disappearing on small screens. If an image is next to text, but the text is not always there... we need to make sure the link always has an acceessible name.

Icon - content - linked

We can do that by, adding aria-label with alternative text to the svg and aria hidden to the span text

For the icon only variation, add a title with alternative text in the svg

Icon - content - linked + text

In the footer there is a twitter icon plus the text of twitter handle next to it all inside of a link. So, the icon is necessary because it provides context.

I was curious if there was any differance between using aria-label on the svg or title inside of it. We already know that, by using title, we get a tooltip on hover and that can be helpful in some cases.

The results were the same in each browser screen reader combo, so either method is acceptable

Icon - content - button

Although the example site did not have a need for buttons, I wanted to test them anyways. So I made these buttons with a cart icon with a plus sign on them. AND similar to the previous example, I wanted to test them the same way using aria-label on the svg or title inside of the svg.

The aria-label version worked everywhere, but the version using title needed to have aria-labelledby on the svg linked to the id on the title element.

SVGs and High Contrast Mode

High Contrast Mode.

Ok, so viewing the site in Widows high contrast mode shows that the logo and all of the icons have disappeared into the black background and need fixed.

On the bright side, since the hero image had a white rectangular box behind the text, it is readable. We should probably take that same approach with the logo (to ensure branding stays intact).

Add a shape to the background

adding a color filled (in this case white) box behind the black text allows us to maintain the desired design of the logo.

currentColor

Unfortunately, currentColor isn't applying the way we expect it to because the spec had changed and browsers updated to line up with it.

Take Aways...

Take aways:

  1. hide decorative images from screen readers aria-hidden="true"
  2. provide meaningful alternative text for content images
  3. add roles to elements to ensure they map properly to the Accessibility API
  4. provide :focus styles for actionable elements
  5. make sure images are not lost in high contrast
